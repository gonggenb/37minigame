# 项目核心约束

## 1. 项目定位

本项目是一款 1 分钟武侠 Roguelite 自动战斗游戏。

核心体验是：

玩家在极短时间内探索地图、遭遇敌人、自动战斗、获得武学和属性成长，然后挑战最终 Boss。

游戏重点不是长线大地图探索，也不是传统回合制 RPG，而是短局压力、路线选择、构筑成长和反复试玩。

## 2. 单局基础流程

单局流程如下：

1. 玩家进入普通地图。
2. 主地图 60 秒倒计时开始。
3. 玩家移动探索地图。
4. 地图中存在敌人、资源点、事件点、隐藏洞穴入口。
5. 玩家触碰普通敌人后进入自动战斗界面。
6. 普通战斗期间，主地图倒计时继续。
7. 战斗胜利后获得奖励，并返回主地图。
8. 玩家可以进入隐藏洞穴。
9. 隐藏洞穴内，主地图倒计时暂停。
10. 60 秒结束后，普通地图阶段结束。
11. 玩家进入最终 Boss 战。
12. Boss 战独立计算，不受 60 秒主地图时间限制。
13. Boss 战结束后进入结算界面。
14. 玩家可以重新开始一局。

## 3. 已确认时间规则

### 3.1 普通地图阶段

普通地图阶段有 60 秒主倒计时。

玩家需要在 60 秒内尽量提升战力，为最终 Boss 战做准备。

### 3.2 普通战斗阶段

玩家触碰普通敌人后进入自动战斗。

普通战斗期间，主地图 60 秒倒计时继续流逝。

这条规则非常重要，因为普通战斗本身就是一种时间消耗。玩家需要判断每场战斗是否值得。

### 3.3 隐藏洞穴阶段

玩家进入隐藏洞穴后，主地图 60 秒倒计时暂停。

隐藏洞穴可以包含战斗、事件、奖励选择或特殊挑战。

初期隐藏洞穴要保持简单：进入洞穴，完成一场战斗或一个事件，获得奖励，返回主地图。

不要一开始把隐藏洞穴做成复杂副本系统。

### 3.4 最终 Boss 阶段

最终 Boss 战不受主地图 60 秒限制。

Boss 战是独立阶段，可以单独统计 Boss 战时间、伤害、胜负和结算评价。

不得把 Boss 战塞回普通 60 秒地图倒计时内。

## 4. 设计目标

项目需要优先满足：

1. 单局短。
2. 节奏快。
3. 自动战斗。
4. 路线选择有压力。
5. 成长反馈明确。
6. 武侠味清晰。
7. 可反复试玩。
8. 系统可以后续扩展。

## 5. 武侠包装方向

推荐术语：

- 生命：气血。
- 能量：真气。
- 经验：修为。
- 技能：武学。
- 被动：心法。
- 高级技能：绝学。
- 货币：铜钱。
- 随机事件：奇遇。
- 隐藏房间：隐藏洞穴、秘境、山洞。
- Boss：强敌、魔头、宗师、守关人。

代码中可以先使用通用英文命名，但 UI 和策划表达应逐步靠近武侠语境。

## 6. 当前目标边界

当前优先做可玩的核心原型。

第一阶段不追求完整内容量、正式美术、商业化、联网或复杂局外养成。

判断一项需求是否该做时，先问它是否能帮助验证以下问题：

- 60 秒主循环是否有压力。
- 碰怪自动战斗是否顺畅。
- 普通战斗继续计时是否形成选择。
- 隐藏洞穴暂停计时是否有吸引力。
- Boss 是否能检验本局成长。

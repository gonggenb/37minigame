# 一分钟武侠 Roguelite 自动战斗游戏策划案

## 1. 项目定位

项目暂名：一分钟江湖

一句话定位：玩家只有 60 秒在江湖地图中碰怪、升级、拿装备、寻找隐藏洞穴，时间结束后进入最终 Boss 自动战斗，用前一分钟的成长结果决定胜负。

核心卖点：

- 60 秒极限成长。
- 碰怪进入自动战斗界面，普通战斗继续消耗主时间。
- 隐藏洞穴暂停主地图时间，提供高风险高收益的额外成长空间。
- 最终 Boss 战不受主 60 秒限制，独立检验本局构筑。

目标体验：

- 玩家每 5 到 10 秒做一次选择。
- 普通战斗是“花时间换成长”。
- 洞穴是“冒风险换额外成长时间”。
- Boss 是“检验这一局是否成型”。

## 2. 核心规则

时间规则：

- 主地图移动时，60 秒倒计时持续减少。
- 玩家在主地图碰到怪物后进入自动战斗界面，战斗期间 60 秒倒计时继续减少。
- 玩家进入隐藏洞穴后，主地图 60 秒倒计时暂停。
- 洞穴内战斗使用同一套自动战斗系统，但不消耗主地图时间。
- 升级、功法三选一、重要奖励选择时暂停所有时间。
- 60 秒归零后进入最终 Boss 战。
- 最终 Boss 战不受主地图 60 秒限制，使用独立 Boss 战时间。

战斗规则：

- 玩家和敌人站在战斗界面两侧。
- 双方根据生命、攻击、防御、攻速、暴击、吸血等属性自动攻击。
- 技能和功法按冷却、攻击次数或条件自动触发。
- 任意一方生命归零后战斗结束。
- 普通战斗失败则本局失败。
- Boss 战失败则本局失败。

## 3. 30 秒核心循环

```text
移动探索 -> 碰到目标 -> 自动战斗/奖励事件 -> 获得经验或装备 -> 升级选择功法 -> 继续寻找更高收益目标
```

60 秒完整单局循环：

```text
进入主地图 -> 60 秒倒计时开始 -> 碰怪自动战斗 -> 获得经验/铜钱/装备 -> 升级三选一 -> 寻找隐藏洞穴 -> 洞穴内暂停主时间并获得高价值奖励 -> 返回主地图 -> 60 秒结束 -> Boss 战 -> 结算
```

## 4. MVP 范围

第一版只做可验证的灰盒原型，不做正式美术、不做商业化、不做联网、不做复杂存档、不做多角色。

P0 必须实现：

- 主时间系统。
- 地图探索系统。
- 遭遇系统。
- 自动战斗系统。
- 怪物系统。
- 奖励系统。
- 升级功法系统。
- 隐藏洞穴系统。
- 最终 Boss 系统。
- 结算系统。
- 配置数据系统。
- 调试系统。

P0 内容量：

- 1 个主地图。
- 1 个玩家角色。
- 3 种普通怪。
- 1 种精英怪。
- 1 种洞穴怪。
- 1 个隐藏洞穴类型。
- 1 个最终 Boss。
- 6 个功法。
- 8 到 12 件临时装备。
- 1 个胜利结算。
- 1 个失败结算。

P0 暂不做：

- 正式美术。
- 多地图。
- 多 Boss。
- 多角色。
- 局外天赋树。
- 商业化。
- 联网排行榜。
- 复杂剧情。

## 5. 系统策划

### 5.1 主时间系统

职责：

- 管理主地图 60 秒倒计时。
- 管理 Boss 独立计时。
- 根据游戏阶段决定时间是否流逝。

状态建议：

```text
MainMapRunning       主地图运行，主时间流逝
NormalBattleRunning  普通战斗，主时间继续流逝
CaveRunning          洞穴阶段，主时间暂停
LevelUpPaused        升级选择，所有时间暂停
BossBattle           Boss 独立战斗
Result               结算
```

配置示例：

```json
{
  "mainTimeLimit": 60,
  "bossTimeLimit": 45,
  "normalBattleConsumesMainTime": true,
  "cavePausesMainTime": true,
  "levelUpPausesAllTime": true
}
```

### 5.2 地图探索系统

职责：

- 控制玩家在主地图移动。
- 生成怪物、宝箱、药草、洞穴入口。
- 让玩家通过碰撞触发遭遇。

地图对象：

- 普通怪：低风险，低收益。
- 精英怪：高风险，高收益。
- 携宝怪：短战斗，特殊奖励。
- 洞穴入口：进入后暂停主时间。
- 宝箱：直接获得装备或铜钱。
- 药草：回复生命。

MVP 地图建议：

- 使用 Unity 2D Top Down 场景。
- 地图尺寸先控制在一屏到两屏半。
- 用灰盒图形或简单 Sprite 占位。
- 对不同目标使用颜色、图标或名字区分风险。

### 5.3 遭遇系统

职责：

- 统一处理玩家碰到对象之后发生什么。
- 将地图对象跳转到战斗、洞穴、奖励、药草或事件。

遭遇类型：

```text
Battle        普通战斗
EliteBattle   精英战斗
Cave          洞穴入口
Treasure      宝箱
Herb          药草
Event         奇遇事件
Boss          Boss 战
```

设计要求：

- 遭遇触发后必须锁定对象，避免重复触发。
- 普通怪和精英怪战斗后从地图移除。
- 洞穴完成后洞穴入口从地图移除或标记已完成。

### 5.4 自动战斗系统

职责：

- 切换到战斗界面。
- 初始化玩家和敌人战斗属性。
- 自动计算双方攻击、技能、伤害、回血和胜负。
- 输出战斗日志供调试。

基础属性：

- 生命。
- 攻击。
- 防御。
- 攻速。
- 暴击率。
- 暴击伤害。
- 吸血。
- 闪避。
- 技能冷却。

伤害公式 MVP：

```text
最终伤害 = max(1, 攻击方攻击 - 防御方防御)
暴击时伤害 = 最终伤害 * 暴击伤害
吸血回复 = 实际造成伤害 * 吸血比例
```

战斗时长目标：

- 普通怪：3 到 5 秒。
- 高血怪：6 到 8 秒。
- 精英怪：8 到 12 秒。
- 洞穴怪：8 到 15 秒。
- 最终 Boss：25 到 45 秒。

### 5.5 怪物系统

职责：

- 通过配置定义怪物属性、类型、风险等级、奖励组。

怪物类型：

- 山贼喽啰：普通怪，低血低攻。
- 野狼：普通怪，低血高攻速。
- 流寇：普通怪，属性均衡。
- 黑风刀客：精英怪，高攻击，掉装备。
- 守洞武人：洞穴怪，高风险，高奖励。

配置字段建议：

```json
{
  "id": "monster_bandit",
  "name": "山贼喽啰",
  "type": "Normal",
  "hp": 40,
  "attack": 6,
  "defense": 1,
  "attackSpeed": 1.0,
  "rewardGroup": "reward_normal_monster",
  "riskLevel": 1
}
```

### 5.6 奖励系统

职责：

- 统一发放经验、铜钱、装备、功法、回血、局外资源。

奖励来源：

- 普通怪。
- 精英怪。
- 洞穴怪。
- 宝箱。
- 药草。
- 升级。
- Boss 通关。

奖励设计：

- 普通怪：经验为主，少量铜钱。
- 精英怪：装备为主，中量经验。
- 洞穴怪：高价值装备或高级功法。
- Boss：胜利结算和局外资源。

### 5.7 升级功法系统

职责：

- 玩家获得经验。
- 经验满后升级。
- 升级时暂停所有时间。
- 弹出三选一功法。
- 功法选择后立即生效。

MVP 功法：

- 剑气诀：每 3 次攻击追加一次 50% 攻击力伤害。
- 疾剑式：攻速提升 15%。
- 铁布衫：最大生命提升 25%，防御提升 2。
- 吸星诀：获得 8% 吸血。
- 毒砂掌：攻击附加中毒，持续造成伤害。
- 破甲掌：攻击降低敌人防御，可叠加少量层数。

设计要求：

- 功法描述必须短。
- 每次升级必须有明显成长。
- 功法效果尽量数据化，便于后续扩展。

### 5.8 装备系统

职责：

- 通过战斗、宝箱、洞穴发放装备。
- 装备后改变玩家属性或提供特殊效果。

MVP 部位：

- 武器。
- 衣服。
- 鞋子。
- 饰品。

MVP 装备示例：

- 青锋剑：攻击 +8，攻速 +5%。
- 血纹刀：攻击 +10，吸血 +5%。
- 旧布衣：生命 +20。
- 铁布衫：生命 +35，防御 +2，攻速 -5%。
- 疾风靴：移动速度 +15%，攻速 +5%。
- 破甲戒：攻击时降低敌人防御。
- 护心玉：Boss 战开始获得护盾。
- 毒镖囊：攻击附加少量毒伤。

MVP 可以不做背包，只做当前装备替换。

### 5.9 隐藏洞穴系统

职责：

- 在主地图生成洞穴入口。
- 玩家进入后暂停主地图 60 秒倒计时。
- 洞穴内触发战斗、宝箱或奖励。
- 完成后返回主地图并恢复主时间。

MVP 洞穴：

- 秘籍洞：击败守洞武人，获得高级功法三选一或装备奖励。

规则：

- 洞穴内死亡则本局失败。
- 洞穴内战斗不消耗主地图时间。
- 洞穴奖励必须明显高于普通怪。
- 每局生成 1 到 2 个洞穴入口。

### 5.10 最终 Boss 系统

职责：

- 主时间归零后强制进入 Boss 战。
- 使用独立 Boss 战计时。
- 胜利进入通关结算，失败进入失败结算。

MVP Boss：黑风寨主

属性方向：

- 高生命。
- 中高攻击。
- 中等防御。
- 低于 50% 生命后攻速提高。

技能：

- 重斩：每隔数秒造成高额伤害。
- 狂暴：半血后攻速提升。
- 召唤喽啰：MVP 可先不做，第二阶段加入。

### 5.11 结算系统

职责：

- 展示胜负。
- 展示本局表现。
- 展示获得奖励。
- 提供重新开始入口。

统计字段：

- 最终等级。
- 完成战斗次数。
- 击败怪物数量。
- 进入洞穴次数。
- 获得装备数量。
- Boss 剩余血量或击杀耗时。
- 本局核心功法。

结算评价示例：

- 你没有进入洞穴，额外成长不足。
- 输出足够，但生存不足。
- Boss 只剩少量生命，下局可优先拿 Boss 增伤。
- 洞穴收益让本局构筑提前成型。

### 5.12 配置数据系统

职责：

- 将主要玩法数值放入 ScriptableObject 或 JSON。
- 支持快速调试战斗时长、奖励、Boss 难度和升级速度。

Unity 推荐：

- MVP 优先使用 ScriptableObject，方便 Inspector 调数值。
- 如果后续需要热更新或表格导入，再补 JSON/CSV 管线。

建议配置资产：

```text
GameBalanceConfig
PlayerConfig
MonsterConfig
BossConfig
KungfuConfig
EquipmentConfig
RewardGroupConfig
CaveConfig
EncounterConfig
```

### 5.13 调试系统

职责：

- 快速验证 60 秒节奏、战斗结果和奖励收益。

MVP 调试功能：

- 设置剩余时间。
- 立即进入 Boss。
- 增加经验。
- 生成普通怪。
- 生成精英怪。
- 生成洞穴。
- 强制战斗胜利。
- 强制战斗失败。
- 打印战斗日志。
- 重新开始本局。

调试 UI 可以只在开发模式显示。

## 6. Unity 技术规划

推荐项目类型：

- Unity 2D URP 或普通 2D。
- 顶视角地图探索。
- 战斗界面使用 Canvas UI 或单独 Battle Scene。
- 第一版优先单机。

推荐场景：

```text
Assets/Scenes/Boot.unity
Assets/Scenes/MainMap.unity
Assets/Scenes/Battle.unity
Assets/Scenes/Cave.unity
Assets/Scenes/BossBattle.unity
Assets/Scenes/Result.unity
```

如果 MVP 追求更快，也可以只用一个 Scene 加状态机切换 UI 面板。

推荐代码结构：

```text
Assets/Scripts/Core
Assets/Scripts/Runtime
Assets/Scripts/Map
Assets/Scripts/Battle
Assets/Scripts/Progression
Assets/Scripts/Data
Assets/Scripts/UI
Assets/Scripts/Debug
```

核心管理器：

- GameFlowController：管理当前阶段。
- GameTimerSystem：管理主时间和 Boss 时间。
- PlayerRuntimeState：保存本局玩家属性、等级、装备、功法。
- EncounterSystem：处理触发遭遇。
- BattleSystem：处理自动战斗。
- RewardSystem：发放奖励。
- LevelUpSystem：处理经验和三选一。
- CaveSystem：处理洞穴进入、完成、返回。
- ResultSystem：生成结算。

## 7. 首轮数值目标

玩家初始：

```text
生命 100
攻击 12
防御 3
攻速 1.0 次/秒
暴击率 5%
暴击伤害 150%
移动速度 适中
```

普通怪参考：

```text
山贼喽啰：生命 35，攻击 5，防御 1，奖励 10 经验
野狼：生命 28，攻击 4，防御 0，攻速较快，奖励 9 经验
流寇：生命 45，攻击 6，防御 1，奖励 12 经验
```

精英怪参考：

```text
黑风刀客：生命 120，攻击 12，防御 3，奖励 30 经验 + 装备
```

洞穴怪参考：

```text
守洞武人：生命 160，攻击 14，防御 4，奖励高级功法或紫色装备
```

Boss 参考：

```text
黑风寨主：生命 550，攻击 18，防御 5，攻速 0.8 次/秒，45 秒 Boss 限时
```

升级参考：

```text
1 -> 2：20 经验
2 -> 3：35 经验
3 -> 4：55 经验
4 -> 5：80 经验
```

目标：普通玩家 60 秒内达到 3 到 5 级，进入 0 到 2 个洞穴，并有机会击败 Boss。

## 8. 验收标准

P0 完成标准：

- 启动游戏后能进入主地图。
- 玩家能移动。
- 主地图 60 秒倒计时正常减少。
- 玩家碰到普通怪后切换到自动战斗。
- 普通战斗期间主倒计时继续减少。
- 战斗胜利后能获得经验并返回主地图。
- 经验满后能升级并弹出三选一功法。
- 升级选择期间时间暂停。
- 玩家能进入隐藏洞穴。
- 洞穴内主地图倒计时暂停。
- 洞穴战斗胜利后能获得高价值奖励并返回主地图。
- 60 秒归零后进入 Boss 战。
- Boss 战使用独立时间。
- Boss 胜利显示通关结算。
- 玩家死亡显示失败结算。
- 主要数值能通过配置调整。
- 有最小调试面板或调试热键。

## 9. 第二阶段建议

P0 跑通后，再做：

- 装备替换 UI。
- 更多洞穴类型：藏宝洞、药王洞、魔窟。
- 随机事件：前辈残魂、石碑悟道、神秘药鼎。
- 商人洞。
- 流派标签：剑气、吸血、毒伤、暴击、防御、Boss 增伤。
- 音效与战斗反馈。
- 局外解锁。
- 新手引导。

# 2.5D 武侠绘本美术规范

> 生产规格、图标尺寸、动画帧规则、提示词、命名、Unity 导入与验收流程统一见
> [《武侠美术生产流水线》](art_production_pipeline.md)。

## 统一方向

项目采用“像素角色 + 低多边形手绘场景”的 2.5D 武侠绘本风格。角色负责动作辨识和战斗反馈，场景负责路线、区域与氛围；两者通过统一色温、哑光材质、克制细节和稳定比例形成同一画面。

## 角色层

- 主角、普通敌人和 Boss 保留像素动画，使用清晰轮廓与较高局部对比度。
- 每帧保持 256 × 256 画布和统一脚底锚点，避免待机、移动、攻击切换时跳动。
- Sprite 使用 Point 过滤；禁止给像素角色添加会模糊边缘的全屏抗锯齿或高光材质。
- Boss 可以比普通敌人更大、更暗，但不能改变像素密度。

## 场景层

- 建筑、树木、岩石与道路保持低多边形轮廓，材质使用哑光、手绘、低反射表现。
- 草地与道路使用世界坐标平铺，缩放长条道路时不拉伸纹理。
- KayKit Medieval 作为当前统一场景基底；新增模型优先匹配其块面、比例和低细节密度。
- 角色始终比场景更锐利、更高对比，避免复杂地面纹理抢夺战斗信息。

## 基础色板

| 用途 | 建议色相 |
| --- | --- |
| 草地 | 灰绿、苔藓绿 |
| 道路 | 暖灰褐、低饱和土色 |
| 岩石与城墙 | 蓝灰、冷灰 |
| 主光 | 柔和暖黄 |
| 环境阴影 | 墨灰绿 |
| UI 强调 | 米白、朱红、暗金 |

## 禁止项

- 不混入写实照片地表、高反射 PBR、霓虹色或高频噪点纹理。
- 不在同一区域混用多个作者差异明显的模型比例和材质体系。
- 不使用带明确方向光影的地表贴图，避免与实时主光冲突。
- 不让环境细节对比度高于角色、敌人与交互提示。

## 当前落地状态

- 草地和道路已经使用同一套世界坐标平铺哑光 Shader。
- 主地图使用统一的暖色主光、灰绿环境光和轻微远景雾。
- 像素角色与 KayKit 低多边形场景保留为有意的 2.5D 混合媒介，而不是完全同材质表现。
- 外部候选资源仍处于筛选状态，未把“找到”写成“已经导入”。

## 执行原则

- 风格规范回答“画成什么样”，生产流水线回答“如何稳定地做出来”。
- 新角色必须先批准种子帧，再一次生成完整动画条带。
- 新图标必须先通过 48 × 48 px 可读性检查，再进入正式 UI。
- 不以单张效果图代替 Unity 游戏内验收。

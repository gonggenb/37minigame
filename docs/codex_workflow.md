# Codex 工作规则

## 1. 基本规则

Codex 每次执行任务前，需要先阅读项目根目录的 `AGENTS.md`。

如果任务涉及玩法实现，还要阅读：

- `docs/project_core.md`
- `docs/gameplay_systems.md`
- `docs/unity_tech.md`

如果任务涉及任务拆解或下一步计划，还要阅读：

- `docs/task_prompts.md`

## 2. 修改原则

每次修改必须优先保证项目可运行、可编译、可测试。

不要为了局部优化破坏核心玩法闭环。

不要擅自重构整个项目。

不要擅自更换引擎、语言、目录结构或主要框架。

不要假装完成了 Unity Editor 中的手动操作。

## 3. 范围控制

如果任务只要求实现一个系统，就只处理该系统相关内容。

如果任务只要求修 Bug，就不要新增玩法。

如果任务只要求整理结构，就不要改变核心规则。

如果确实需要调整范围，必须说明原因和影响。

## 4. 禁止默认接入

除非用户明确要求，否则不要接入：

- 真实广告 SDK。
- 真实支付 SDK。
- 后端服务。
- 账号系统。
- 联机系统。
- 排行榜系统。
- 云存档。
- 大型第三方框架。

可以做接口预留或文档说明，但不要实际接入。

## 5. 核心规则验证

每次涉及流程、战斗、计时、地图、洞穴、Boss 的修改后，都必须验证：

- 普通战斗期间，主地图 60 秒倒计时是否继续。
- 隐藏洞穴内，主地图 60 秒倒计时是否暂停。
- Boss 战是否独立于主地图 60 秒倒计时。

## 6. Unity Editor 操作说明

如果某项功能需要 Unity Editor 手动设置，完成回复必须说明：

- 需要创建的 GameObject。
- 需要挂载的脚本。
- Inspector 需要绑定的字段。
- 需要拖入的 Prefab、UI 或配置引用。
- 推荐的 Play Mode 验证步骤。

不要用“已绑定”“已配置”等措辞描述 Codex 无法实际完成的 Editor 操作。

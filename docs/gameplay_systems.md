# 玩法系统拆分

## 1. 系统拆分原则

不要把所有逻辑写在一个大脚本里。

按功能拆分系统，每个系统只负责自己的职责。

系统实现必须服务于最小核心闭环，避免一开始堆大量扩展玩法。

## 2. 游戏流程系统

负责单局状态切换。

至少包括：

- 准备阶段。
- 普通地图阶段。
- 普通战斗阶段。
- 隐藏洞穴阶段。
- Boss 战阶段。
- 结算阶段。

必须明确区分：

- 主地图 60 秒倒计时。
- Boss 战独立计时。

建议状态：

```text
Ready
MainMapRunning
NormalBattleRunning
CaveRunning
LevelUpPaused
BossBattle
Result
```

## 3. 主地图探索系统

负责普通地图中的移动、探索、敌人、资源点、事件点和隐藏洞穴入口。

主地图阶段必须围绕 60 秒倒计时设计。

地图不需要一开始很大，优先保证节奏密度和可玩闭环。

地图对象可以包括：

- 普通敌人。
- 精英敌人。
- 宝箱。
- 药草或恢复点。
- 奇遇事件点。
- 隐藏洞穴入口。

## 4. 遭遇系统

负责判断玩家触碰了什么对象，并触发对应玩法。

包括：

- 普通敌人。
- 精英敌人。
- 资源点。
- 宝箱。
- 事件点。
- 隐藏洞穴入口。
- Boss 入口，若后续需要。

普通敌人遭遇后进入自动战斗。

隐藏洞穴入口遭遇后进入隐藏洞穴阶段。

遭遇触发后要防止重复触发。

## 5. 自动战斗系统

负责双方自动攻击、技能释放、胜负判定、战斗日志和奖励触发。

普通战斗规则：

- 玩家碰到怪物后进入战斗界面。
- 双方自动攻击。
- 直到一方落败。
- 战斗期间主地图倒计时继续。
- 玩家胜利后返回普通地图。
- 玩家失败后进入失败结算。

Boss 战规则：

- Boss 战是独立阶段。
- Boss 战不受主地图 60 秒限制。
- 可以单独统计 Boss 战时间。
- Boss 战结束后进入结算。

基础属性至少支持：

- 气血。
- 攻击。
- 防御。
- 攻速。
- 暴击率。
- 暴击伤害。
- 吸血。
- 闪避。

## 6. 角色属性系统

至少支持：

- 气血。
- 攻击。
- 防御。
- 攻速。
- 暴击率。
- 暴击伤害。
- 闪避。
- 移速。
- 伤害加成。
- 减伤。
- 恢复。

属性来源后续可以包括：

- 武学。
- 心法。
- 装备。
- 奇遇。
- 洞穴奖励。
- 临时 Buff。

## 7. 武学系统

武学是本项目的核心构筑内容。

可分为：

- 外功：攻击、连击、范围伤害。
- 内功：气血、防御、恢复、减伤。
- 身法：移速、闪避、攻速。
- 心法：被动规则和属性转换。
- 绝学：高稀有度核心能力。

当前阶段优先做少量清晰武学，不要一次性做大量复杂技能。

升级三选一时必须暂停所有时间。

## 8. 奖励系统

负责战斗奖励、地图拾取、洞穴奖励、事件奖励、Boss 奖励和结算奖励。

奖励可以包括：

- 铜钱。
- 修为。
- 武学秘籍。
- 临时属性。
- 药品。
- 宝箱。
- 装备。
- 局外解锁资源。

当前优先做局内奖励闭环，不要过早做复杂长线养成。

## 9. 隐藏洞穴系统

隐藏洞穴定位是高价值额外内容。

核心规则：

- 进入洞穴后，主地图 60 秒倒计时暂停。
- 洞穴内可以发生战斗、奇遇或奖励选择。
- 洞穴结束后返回主地图。
- 返回主地图后，主地图倒计时继续。

初期洞穴不要做成复杂副本。

推荐 MVP 流程：

```text
发现洞穴入口 -> 进入洞穴 -> 主时间暂停 -> 触发一场战斗或奖励事件 -> 获得奖励 -> 返回主地图 -> 主时间恢复
```

## 10. Boss 系统

Boss 是单局最终验证点。

至少支持：

- Boss 属性。
- Boss 普攻。
- Boss 技能，初期可简单。
- 玩家自动攻击。
- 胜负判定。
- Boss 战计时。
- 结算入口。

Boss 战不得消耗主地图 60 秒倒计时。

## 11. UI 系统

普通地图 HUD 至少显示：

- 主地图剩余时间。
- 玩家气血。
- 当前战力或关键属性。
- 当前主要武学。
- 当前铜钱或修为。

战斗界面至少显示：

- 玩家血量。
- 敌人血量。
- 当前战斗状态。
- 伤害反馈。
- 胜负结果。

结算界面至少显示：

- 是否击败 Boss。
- 击杀数量。
- 获得奖励。
- 本局关键武学。
- 重新开始按钮。

## 12. 配置系统

以下内容应尽量配置化：

- 玩家基础属性。
- 敌人属性。
- Boss 属性。
- 武学效果。
- 奖励内容。
- 地图生成。
- 洞穴内容。
- 随机事件。

不要把大量数值写死在逻辑代码里。

## 13. 存档系统

当前阶段只需要轻存档，甚至可以暂缓。

可记录：

- 已解锁武学。
- 局外资源。
- 历史最高击杀。
- 历史最快通关。
- 设置项。
- 新手引导进度。

不要过早做账号、云存档或服务器存档。

## 14. Debug 系统

建议保留调试功能：

- 立即开始一局。
- 重置本局。
- 增加玩家攻击。
- 增加玩家气血。
- 增加经验或修为。
- 生成普通敌人。
- 生成隐藏洞穴。
- 立即进入 Boss。
- 强制胜利。
- 强制失败。
- 打印当前属性。
- 打印战斗日志。
- 打印当前游戏状态。

Debug 功能不得破坏正式玩法逻辑。

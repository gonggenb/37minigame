# 一分钟武侠 Roguelite 自动战斗游戏 v0.1 策划案

## 1. 版本目标

v0.1 的目标不是做完整商业游戏，而是先验证一个可以反复试玩的 60 秒核心闭环。

一句话体验：

玩家扮演初入江湖的少侠，在 60 秒内跑图、碰怪、自动战斗、拿武学、找隐藏洞穴，最后用这一分钟积累出的构筑挑战黑风寨主。

本版本必须回答四个问题：

1. 60 秒主地图是否有紧张感。
2. 普通战斗继续消耗主时间是否会形成取舍。
3. 隐藏洞穴暂停主时间是否足够有吸引力。
4. Boss 战是否能检验本局成长结果。

## 2. 核心体验支柱

### 2.1 短局压力

单局主地图只有 60 秒。玩家不能清空地图，只能在有限时间内选择更划算的目标。

设计关键词：

- 每 5 到 10 秒产生一次选择。
- 普通怪是稳定成长。
- 精英怪是高耗时高收益。
- 洞穴是高价值机会点。
- Boss 是最终检验。

### 2.2 自动战斗

战斗不要求玩家操作连招，重点是进入战斗前的路线选择和战斗后的构筑成长。

普通战斗的意义不是操作难度，而是时间成本：

- 打弱怪：更快、更稳、收益低。
- 打精英：更慢、更危险、收益高。
- 进洞穴：主时间暂停，但战斗失败仍会输掉本局。

### 2.3 武侠构筑

玩家本局成长主要来自武学、心法、装备和洞穴奖励。

v0.1 优先做少量但差异清晰的构筑方向：

- 剑气连击：依靠攻速和追加伤害。
- 铁布防御：依靠气血、防御和减伤。
- 吸血续航：依靠攻击和吸血。
- 毒伤消耗：依靠持续伤害打 Boss。
- 破甲爆发：降低敌人防御后打高伤害。

## 3. 单局流程

```text
开始一局
-> 进入主地图
-> 60 秒倒计时开始
-> 玩家移动探索
-> 碰到普通怪 / 精英怪 / 宝箱 / 药草 / 洞穴入口
-> 普通怪和精英怪进入自动战斗，主倒计时继续
-> 胜利获得修为、铜钱、装备或武学
-> 修为满时升级，弹出三选一，所有时间暂停
-> 进入隐藏洞穴时主倒计时暂停
-> 洞穴战斗或事件结束后返回主地图，主倒计时继续
-> 60 秒归零
-> 进入最终 Boss 战
-> Boss 战独立计时，不受 60 秒限制
-> 胜利或失败后进入结算
-> 重新开始下一局
```

## 4. 三条硬时间规则

以下规则是 v0.1 的验收红线：

1. 普通地图碰怪战斗时，主地图 60 秒倒计时继续流逝。
2. 隐藏洞穴内战斗时，主地图 60 秒倒计时暂停。
3. 最终 Boss 战不受 60 秒限制，作为独立阶段单独计算 Boss 战时间。

升级三选一、重要奖励选择、暂停菜单可以暂停所有时间。

## 5. v0.1 内容范围

### 5.1 必须做

- 1 个主测试场景。
- 1 个可移动玩家。
- 60 秒主地图倒计时。
- 普通怪碰撞触发自动战斗。
- 普通战斗期间主时间继续减少。
- 战斗胜利后获得修为和奖励。
- 升级三选一武学。
- 1 种隐藏洞穴入口。
- 洞穴内暂停主时间。
- 1 个最终 Boss。
- Boss 战独立计时。
- 胜利、失败、重新开始。
- 最小可用 HUD 和调试功能。

### 5.2 暂不做

- 真实广告 SDK。
- 真实支付 SDK。
- 后端、账号、联机、排行榜、云存档。
- 大型剧情。
- 大型装备词条系统。
- 大规模正式美术接入。
- 多角色、多地图、多 Boss。
- 复杂局外养成。

## 6. 地图设计

v0.1 使用小型顶视角主地图，尺寸控制在一屏到两屏半。重点不是迷宫，而是目标密度和路线选择。

地图对象：

| 对象 | 作用 | 设计目标 |
| --- | --- | --- |
| 普通怪 | 稳定修为来源 | 3 到 5 秒解决 |
| 精英怪 | 高收益风险目标 | 8 到 12 秒解决 |
| 宝箱 | 直接给装备或铜钱 | 鼓励绕路 |
| 药草 | 回复气血 | 给残血玩家一个选择 |
| 隐藏洞穴 | 暂停主时间的高价值内容 | 每局 1 到 2 个 |

主地图需要让玩家快速判断目标价值。早期可以用颜色和文字区分：

- 灰色：普通怪。
- 红色：精英怪。
- 金色：宝箱。
- 绿色：药草。
- 紫色：隐藏洞穴。

## 7. 战斗设计

### 7.1 基础属性

玩家和敌人至少包含：

- 气血。
- 攻击。
- 防御。
- 攻速。
- 暴击率。
- 暴击伤害。
- 吸血。
- 闪避。
- 移速。

### 7.2 MVP 伤害公式

```text
基础伤害 = max(1, 攻击方攻击 - 防御方防御)
暴击伤害 = 基础伤害 * 暴击伤害倍率
吸血回复 = 实际伤害 * 吸血比例
```

闪避成功时本次伤害为 0。

### 7.3 战斗目标时长

| 战斗类型 | 目标时长 |
| --- | --- |
| 普通怪 | 3 到 5 秒 |
| 高血普通怪 | 6 到 8 秒 |
| 精英怪 | 8 到 12 秒 |
| 洞穴怪 | 8 到 15 秒 |
| Boss | 25 到 45 秒 |

## 8. 首批敌人与 Boss

| 名称 | 类型 | 定位 | 建议数值 |
| --- | --- | --- | --- |
| 山贼喽啰 | 普通怪 | 低风险样本 | 气血 35，攻击 5，防御 1 |
| 野狼 | 普通怪 | 快攻低血 | 气血 28，攻击 4，防御 0，攻速较快 |
| 流寇 | 普通怪 | 均衡样本 | 气血 45，攻击 6，防御 1 |
| 黑风刀客 | 精英怪 | 高攻击高收益 | 气血 120，攻击 12，防御 3 |
| 守洞武人 | 洞穴怪 | 洞穴挑战 | 气血 160，攻击 14，防御 4 |
| 黑风寨主 | 最终 Boss | 构筑检验 | 气血 550，攻击 18，防御 5，攻速 0.8 |

Boss 技能 v0.1 保持简单：

- 重斩：每隔数秒造成一次高额伤害。
- 狂暴：气血低于 50% 后攻速提升。

召唤小怪、复杂技能圈和多阶段演出放到后续版本。

## 9. 首批武学

| 名称 | 类型 | 效果 | 构筑用途 |
| --- | --- | --- | --- |
| 剑气诀 | 外功 | 每 3 次攻击追加一次 50% 攻击伤害 | 连击输出 |
| 疾剑式 | 身法 | 攻速提升 15% | 高频触发 |
| 铁布衫 | 内功 | 最大气血提升 25%，防御 +2 | 生存 |
| 吸星诀 | 心法 | 获得 8% 吸血 | 续航 |
| 毒砂掌 | 外功 | 攻击附加中毒，持续造成伤害 | 长战斗和 Boss |
| 破甲掌 | 外功 | 攻击降低敌人少量防御，可叠加 | 后期爆发 |

升级三选一规则：

- 弹出选择时暂停所有时间。
- 每次出现 3 个不同武学或升级项。
- 选择后立即生效。
- 描述必须短，玩家一眼能看懂收益。

## 10. 奖励设计

奖励优先服务 60 秒内的成长反馈。

| 来源 | 奖励倾向 |
| --- | --- |
| 普通怪 | 修为为主，少量铜钱 |
| 精英怪 | 中量修为，较高概率装备 |
| 宝箱 | 装备、铜钱或特殊武学 |
| 药草 | 回复气血 |
| 洞穴 | 高级武学、强装备或大量修为 |
| Boss | 胜利结算和后续局外解锁预留 |

v0.1 可以不做完整背包。装备可以采用“获得后直接替换或直接转为属性”的简化方案。

## 11. 首批数值目标

玩家初始建议：

| 属性 | 数值 |
| --- | --- |
| 气血 | 100 |
| 攻击 | 12 |
| 防御 | 3 |
| 攻速 | 1.0 次/秒 |
| 暴击率 | 5% |
| 暴击伤害 | 150% |
| 吸血 | 0% |

升级经验建议：

| 等级 | 所需修为 |
| --- | --- |
| 1 -> 2 | 20 |
| 2 -> 3 | 35 |
| 3 -> 4 | 55 |
| 4 -> 5 | 80 |

60 秒目标结果：

- 普通玩家能达到 3 到 5 级。
- 普通玩家能完成 6 到 10 场普通战斗。
- 普通玩家有机会进入 0 到 2 个洞穴。
- 构筑合理时可以击败 Boss。
- 不进洞穴也有小概率赢，但难度明显更高。

## 12. UI 需求

### 12.1 主地图 HUD

必须显示：

- 主地图剩余时间。
- 玩家气血。
- 当前攻击或战力。
- 当前等级或修为。
- 当前主要武学。

### 12.2 战斗界面

必须显示：

- 玩家气血。
- 敌人气血。
- 敌人名称。
- 当前战斗状态。
- 胜利或失败结果。

普通战斗界面要能看出主时间仍在流逝。

### 12.3 洞穴界面

必须显示：

- 当前处于隐藏洞穴。
- 主地图时间已暂停。
- 洞穴奖励或洞穴敌人信息。

### 12.4 Boss 界面

必须显示：

- Boss 名称。
- Boss 气血。
- 玩家气血。
- Boss 战独立计时或已用时。

不要显示主地图 60 秒倒计时继续减少。

### 12.5 结算界面

必须显示：

- 胜利或失败。
- 最终等级。
- 击杀数量。
- 进入洞穴次数。
- 获得的关键武学。
- Boss 战耗时或 Boss 剩余血量。
- 重新开始按钮。

## 13. Unity 落地建议

v0.1 建议使用单场景加状态机切换 UI 面板，减少场景切换复杂度。

推荐场景：

```text
Assets/Scenes/MainPrototype.unity
```

推荐脚本职责：

| 脚本 | 职责 |
| --- | --- |
| GameFlowController | 单局阶段切换 |
| MainTimerController | 主地图 60 秒倒计时 |
| PlayerController | 玩家移动 |
| PlayerStats | 玩家本局属性 |
| EncounterManager | 处理碰撞遭遇 |
| BattleManager | 自动战斗和胜负 |
| RewardManager | 发放修为、铜钱、装备、武学 |
| MartialArtManager | 武学选择和生效 |
| CaveManager | 洞穴进入、完成、返回 |
| BossBattleManager | Boss 战独立逻辑 |
| HUDController | 主 HUD、战斗面板、结算面板 |
| DebugPanelController | 调试按钮或快捷键 |

配置优先使用 ScriptableObject 或集中配置类。不要把大量数值散落写死在各个 MonoBehaviour 中。

## 14. 调试功能

v0.1 至少保留：

- 开始一局。
- 重置本局。
- 生成普通怪。
- 生成精英怪。
- 生成隐藏洞穴。
- 增加修为。
- 增加玩家攻击。
- 增加玩家气血。
- 直接进入 Boss。
- 强制当前战斗胜利。
- 强制当前战斗失败。
- 打印当前游戏阶段、主时间、Boss 时间和玩家属性。

调试功能只服务开发验证，不参与正式玩法奖励。

## 15. 验收标准

v0.1 可试玩闭环完成时，必须满足：

- 启动后能进入主地图。
- 玩家能移动。
- 主地图 60 秒倒计时正常减少。
- 碰到普通怪后进入自动战斗。
- 普通战斗期间主倒计时继续减少。
- 普通战斗胜利后能返回主地图。
- 普通战斗失败后进入失败结算。
- 战斗胜利后能获得修为。
- 修为满后能升级并三选一。
- 三选一期间所有时间暂停。
- 玩家能进入隐藏洞穴。
- 洞穴阶段主地图倒计时暂停。
- 洞穴结束后返回主地图并恢复主时间。
- 60 秒归零后进入 Boss 战。
- Boss 战不受主地图 60 秒限制。
- Boss 战有独立计时。
- Boss 胜利进入胜利结算。
- Boss 失败进入失败结算。
- 结算后可以重新开始。

## 16. 后续版本方向

P0 闭环跑通后，再按优先级追加：

1. 更多武学和流派标签。
2. 装备替换 UI。
3. 更多洞穴类型，例如藏宝洞、药王洞、魔窟。
4. 奇遇事件，例如石碑悟道、前辈残魂、神秘药鼎。
5. 战斗音效和伤害反馈。
6. 简单局外解锁。
7. 新手引导。
8. 美术替换和 UI 美化。

商业化、联网、排行榜、账号和云存档不属于 v0.1 范围。
